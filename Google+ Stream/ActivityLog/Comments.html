Commented on post by Greg PerkinsI'm an emacs user because when I started my first job I had no editor of choice, and some very smart coworkers told me, "Use emacs." So I did. I think I would be equally happy had I started on vi/vim. I'm not that religious about emacs vs. vi, but I am very happy that my primary editor is one that can easily be installed in almost any terminal. That's the key for me - no mucking around with NFS or SMB mounts, just hop on a box and edit files right through SSH. Particularly in an environment when you're jumping around between different boxes frequently, I couldn't imagine trying to mount them as local file systems first. However, I have experienced the bliss that is Visual Studio when hacking on a C# project once, and definitely appreciate the draw to and potential efficiency gains of using an editor that's even slightly more robust than emacs if not a full blown IDE. I suppose I see that as a different use case though. Developing applications and editing files aren't necessarily synonymous. You're always going to find yourself needing to muck around with /etc/ssh/ssh_config, ~/.bashrc, etc. at some point, so knowing a terminal-based editor is good, even if it's not always ideal for what you do 95% of the time if you're an application developer.</rumination> — I'm really starting to love Sublime Text. I figure I should learn emacs in parallel, though, just to maintain a hardcore facade.
